# OWASP Juice Shop's Injection Points

## Endpoint 1: Product Search
*How to Find Out:* Scanning tools such as ZAP, SQLmap

The vulnerable functionality is accessible through the following API endpoint:
```
GET /rest/products/search?q=
```
The q parameter is used as the search keyword to filter products in the database.

Example request:

<img width="957" height="465" alt="image" src="https://github.com/user-attachments/assets/302d1749-6dc5-4c73-a1fc-b15a3ecddc2a" />

The backend query generated by this request is logically equivalent to:

```
SELECT * FROM Products
WHERE ((name LIKE '%test%' OR description LIKE '%test%')
AND deletedAt IS NULL)
ORDER BY name;
```

Because the user-supplied parameter is directly concatenated into the SQL query, malicious input can alter the query structure and lead to SQL Injection.


## Endpoint 2: Login API
*How to Find Out:* Testing by hand =)))

The application’s authentication system is implemented as a REST API instead of a traditional HTML login form.

Login requests are sent to the following endpoint:
```
POST /rest/user/login
Content-Type: application/json
```
Request body:
```
{
 "email": "test@test.com",
 "password": "123456"
}
```

<img width="960" height="463" alt="image" src="https://github.com/user-attachments/assets/d4c4e967-5924-44f6-8234-ed53ba69c130" />

Because the parameters are transmitted within a JSON body rather than standard form parameters, some automated scanners may fail to properly fuzz the input fields. Therefore, manual testing was performed to verify whether the endpoint is vulnerable to SQL Injection.


## Endpoint 3: Review Function
*How to Find Out:* By reviewing source code

After cloning the OWASP Juice Shop repository locally, open the project in a tool that supports searching across files (e.g., VSCode, which is used in this report). Navigate to Edit → Find and search for keywords such as “review” or “product review”.

From the search results, focus on the file showProductReviews.ts, particularly the following line:
```
db.reviewsCollection.find({ $where: 'this.product == ' + id })
```
